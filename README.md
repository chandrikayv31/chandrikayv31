### 🧠 What I actually work with

**LLM & Agentic AI**
Building autonomous agents isn't just prompt engineering — it's orchestration. I work with LangChain and LangGraph for multi-agent workflows, Semantic Kernel for tool-calling and memory, and MCP (Model Context Protocol) to give agents governed, real-time access to enterprise systems instead of static context windows.

**RAG & Retrieval**
I've built RAG pipelines that go beyond "stuff it in a prompt" — vector indexing across FAISS, Pinecone, and Weaviate, paired with custom evaluation scoring for groundedness, relevance, and factual accuracy before anything reaches production.

**MLOps & Deployment**
Models don't matter until they're serving traffic reliably. I use MLflow for lifecycle tracking, Docker/Kubernetes for deployment, and Cloud Build for CI/CD — with structured output validation so agent responses hit downstream systems as clean, typed data instead of loose text.

**Classical ML**
Before GenAI, I spent years on gradient boosting (XGBoost, LightGBM), feature engineering, and anomaly detection — skills that still show up when I'm debugging why an agent's decision doesn't match the underlying data.

### 🚀 Things I've built

**Agentic Network Remediation System** — T-Mobile
Multi-agent pipeline where separate agents handle root cause analysis, constraint validation, and execution — cutting MTTR on network and billing incidents. Agents draft and dry-run their own remediation scripts before anything touches production, cutting manual automation work by ~60%.

**Clinical NLP & Risk Prediction** — UPMC
Pulled structured signal (diagnoses, risk indicators) out of unstructured physician notes, and built a high-risk patient outcome model running at ~78% precision. Every pipeline was HIPAA-compliant with full audit trail design — because in healthcare, "the model is right" isn't enough, you have to prove it.

**Credit Risk Scoring Engine** — TCS
Built and shipped an XGBoost credit risk model on ~300K records that cut default rates ~14% in three months. Engineered 50+ features from raw transaction data — repayment velocity and bureau enquiry patterns moved the needle more than any model tuning did.
